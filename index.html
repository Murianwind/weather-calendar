원본 저장소: github.com/Murianwind/weather-calendar

1 프로젝트 포크 (Fork)

상단 원본 저장소 링크 접속 후 오른쪽 상단의 [Fork] 버튼을 눌러 내 계정으로 가져옵니다.

🔄 기존 포크 사용자 업데이트 방법

이미 포크하신 분들은 본인 저장소 메인에서 [Sync fork] → [Update branch]를 차례로 클릭하세요. 최신 코드가 반영되어 더 이상 파일을 직접 수정하지 않고 Secrets 설정만으로 관리할 수 있습니다.

2 기상청 API 인증키 발급

1. 기상청 API 허브 회원가입 후 [인증키 현황]에서 키를 복사합니다.

2. 메뉴에서 단기예보, 중기기온, 중기육상예보를 각각 활용 신청하세요.

4 GitHub Secrets 설정

내 저장소 Settings → Secrets and variables → Actions에서 아래 항목들을 각각 New repository secret으로 추가합니다.

  • • KMA_API_KEY : 기상청 인증키
  • • KMA_NX : 위에서 찾은 NX 값
  • • KMA_NY : 위에서 찾은 NY 값
  • • REG_ID_LAND : 위에서 찾은 LAND 값
  • • REG_ID_TEMP : 위에서 찾은 TEMP 값
  • • LOCATION_NAME : 내 동네 이름 (예: 우리집)

5 자동 업데이트 활성화

⚠️ 중요: 워크플로우 활성화
GitHub 정책상 Fork된 저장소는 자동 실행이 꺼져 있습니다.
상단 [Actions] 탭 클릭 후 "I understand my workflows, go ahead and enable them"을 눌러 활성화하세요.
⏳ 첫 실행 방법
Actions 탭에서 Update Weather ICS 선택 후 Run workflow를 클릭하면 즉시 캘린더 파일이 생성됩니다.

6 캘린더 구독하기

1. 내 저장소 메인에 생성된 weather.ics 파일을 클릭합니다.
2. [Raw] 버튼을 우클릭하여 주소를 복사합니다.
3. 아이폰(캘린더 앱) 또는 구글 캘린더의 [URL로 구독] 메뉴에 붙여넣으세요.

⏰ 업데이트 주기 안내
  • 단기 예보 (오늘~3일): 매 3시간마다 최신 정보로 자동 갱신됩니다.
  • 중기 예보 (4일~10일): 하루 2번(06시, 18시) 최신 정보로 자동 갱신됩니다.

※ 실제 앱 반영 속도는 서비스별 갱신 주기에 따라 다를 수 있습니다.
